Employee Retention - Data Cleaning & Feature Engineering

The Breakdown¶

In this module, part 2 of 4, we're going to combine the Data Cleaning and Feature Engineering for this project. This is important because, as a general rule, better data beats better algorithms.

In this module, I'm going to cover the essential steps for building an analytical base table (ABT):

  1. Drop unwanted observations
  2. Fix structural errors
  3. Handle missing data
  4. Engineer features
  5. Save the ABT

Finally, I'll save the ABT to a new file so I can use it in other modules.

1. Drop unwanted observations¶

The first step to data cleaning is removing samples from your dataset that you don't want to include in the model. For this data set this means removing all duplicates and dropping any classes that aren't pertinent to the problem. If you recall from Part 1, there were a lot of sub-categories for the 'department' feature. I printed them out below to inspect them and make sure they were all pertinent to the problem I'm trying to solve. As we're only concerned with the behavior of full-time, permanent workers I decided to remove all observations belonging to the 'temp' department. Note that I printed the dataframe shape after each drop to verify it was successful. I prefer to double check my work as I go so I don't compound errors and oversights as I go through my analysis.

2. Fix structural errors¶

The next bucket under data cleaning involves fixing structural errors, which arise during measurement, data transfer, or other types of "poor housekeeping". Back in the Exporatory Analysis I noted that both 'filed_complaint' and 'recently_promoted' had lots of missing data and may need to be categorical variables. I also noted that there was a categorization error with both 'IT' and 'Information Technology' being 'department' subcategories. Well here's where I fix all those structural errors.

Notice again how I verify the changes I'm making come out as I expect as I go through this step. I'm printing out the unique values of the 'filed_complaint' and 'recently_promoted' errors as well as recreating the 'department' bar graph to make sure my changes stick and no other categorical errors stick out to me. An ounce of prevention is worth a pound of cure.

3. Handle missing data¶

Next, it's time to handle missing data. As a general rule, I avoid simply deleting or interpolating missing data as that fundamentally changes the data set in a way that may obscure valuable information. Missingness is meaningful. So instead I prefer to label missing data as such. Then when it shows up in further analysis I can decide what that missing data may be trying to tell me. For the 'last_evaluation' missing data I converted the NaN values to 0 so the data can be useful to my modeling algoritms. I also created an indicator variable of missingness using the '.isnull()' method, 'last_evaluation_missing' that I suspect my be useful later on. At the end I double check to make sure I'm no longer missing any data before moving on to Feature Engineeing.

4. Engineer features¶


Do you remember the scatterplot of 'satisfaction' and 'last_evaluation' for employees who have 'Left'? I've reproduced that figure below for reference. In it, we can see the three distinct groupings. I'm going to use boolean masks to create indicator features for these three groups as follows:

  • 'underperformer' - last_evaluation < 0.6 and last_evaluation_missing == 0
  • 'unhappy' - satisfaction_level < 0.2
  • 'overachiever' - last_evaluation > 0.8 and satisfaction > 0.7

Next, I want to convert 'status' into an indicator variable where 'Left' equals 1 and 'Employed' equals 0 using the pd.get_dummies() method.

Finally, I want to create dummy variables for all my categorical features as well. To do this concisely I will pass a feature list into my pd.get_dummies function. Once I have created all my dummy variables I double check by pulling the .mean() method and overwriting my dataframe with all the new dummy variables and checking the .head() to make sure they're all there before moving on to the final step of saving the new dataframe as my analytical base table for use in my modeling in the next section.
